## リスクと対応策

| リスク | 対応策 |
|-------|--------|
| **Gemini API の遅延・障害** | ダミーレスポンスで代替、スコア計算ロジックのテストを先行 |
| **売上データが取得できない** | ダミーデータで代替 |
| **AWS 移行に手間取る** | 最悪EC2に全乗せ。それも無理ならローカルで動作しているところを見せる |

## 成功定義

| KPI | 目標値 |
|-----|--------|
| **完成度** | 全機能が本番環境で動作 |
| **テーマ・技術活用** | Gemini + リアルタイム画像認識の実装 |
| **面白さ・独自性** | 新しい買い物体験の提示 |
| **デモ成功率** | 何回か撮ってみて認識精度を定性評価(定量評価基盤を整えられればさらにgood) |
